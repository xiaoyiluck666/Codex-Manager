#ТЕСТИРОВАНИЕ

Этот документ задаёт базовые требования к тестированию и валидации на уровне репозитория CodexManager.

Цель:

- Новые сотрудники могут быстро узнать, что нужно запустить, по крайней мере, после внесения изменений.
– Избегайте обновления всех изменений до полной проверки.
- Обеспечьте фиксированные входы для проверки изменений высокого риска, таких как совместимость протоколов, публикация ссылок и управление настройками.

## 1. Базовая среда

-Node.js 20
- пнпм 9
- Устойчивость к ржавчине
- PowerShell 7+ (Windows Проверка упаковки/скрипта)

## 2. Изменения во внешнем интерфейсе

Область применения:

- `apps/src/app/`
- `apps/src/components/`
- `apps/src/lib/`
- `apps/src/hooks/`

Минимальная проверка:

``` баш
pnpm -C приложения запускают сборку
pnpm -C приложения запускают тест: время выполнения
```

Описание:

- `pnpm -C apps run build`: убедитесь, что ссылка статического экспорта Next.js все еще работает.
- `pnpm -C apps run test:runtime`: Подтвердите определение возможностей во время выполнения и рабочий стол / Web Логика ухудшения возможностей не вернулась
- Если изменения включают идентификацию во время выполнения, Web RPC, обработку различий рабочего стола/Web, составьте Раздел 4.

## 3. Рабочий стол / Tauri Изменения

Область применения:

- `apps/src-tauri/`
- Обновления рабочего стола, лоток, окно, изменения, связанные с командным мостом.

Минимальная проверка:

``` баш
грузовой тест --workspace
```

Дополнительные предложения:

```powershell
pwsh -NoLogo -NoProfile -File scripts/rebuild.ps1 -Bundle nsis -CleanDist
```

Описание:

- Всякий раз, когда вы меняете Tauri мост или жизненный цикл рабочего стола, рекомендуется хотя бы один раз выполнить проверку сборки локального рабочего стола.

## 4. Web Запуск изменений, совместимых с оболочкой/развертыванием

Область применения:

- `crates/web/`
- `apps/src/lib/api/transport.ts`
- `apps/src/components/layout/app-bootstrap.tsx`
- `apps/src/components/layout/header.tsx`
- `apps/src/components/layout/sidebar.tsx`
- Web Агент, `/api/runtime`, `/api/rpc` и соответствующие изменения в методах развертывания.

Минимальная проверка:

``` баш
pnpm -C приложения запускают сборку
pnpm -C приложения запускают тест: время выполнения
грузовой тест -p codexmanager-web
pwsh -NoLogo -NoProfile -File scripts/tests/web_runtime_probe.test.ps1
```

Предлагаемые дополнения:

```powershell
pwsh -NoLogo -NoProfile -File scripts/tests/web_runtime_probe.ps1 `
  -База http://localhost:48761
pwsh -NoLogo -NoProfile -Файл скриптов/тестов/web_ui_smoke.ps1 -SkipBuild
```

Описание:

- `pnpm -C apps run build`: убедитесь, что внешний статический экспорт все еще может быть создан.
- `pnpm -C apps run test:runtime`: убедитесь, что контракт среды выполнения внешнего интерфейса и определение возможностей согласованы.
- `cargo test -p codexmanager-web`: Подтвердить Web маршрутизацию оболочки и контракт на проверку времени выполнения.
- `web_runtime_probe.test.ps1`: Подтвердите поведение сценария Web для запуска оболочки с минимальной дымовой ссылкой.
- `web_ui_smoke.ps1`: подтверждение ключевого поведения пользовательского интерфейса страниц Web в поддерживаемых/неподдерживаемых оболочках запуска.

Область применения:

- `crates/service/src/gateway/`

Должно охватывать:

- `/v1/chat/completions`
- Потоковая передача SSE
- Непоточный JSON
- `tools`
- `tool_calls`

Минимальная проверка:

``` баш
грузовой тест --workspace
pwsh -NoLogo -NoProfile -Файл scripts/tests/gateway_reгрессия_suite.ps1
pwsh -NoLogo -NoProfile -Файл скриптов/тестов/codex_stream_probe.ps1

```

Описание:

- Если в локальной среде нет реальной вышестоящей учетной записи, по крайней мере, запустите тест Rust и сохраните инструкции по выполнению проверки.
- Исправление совместимости, из-за которого невозможно было проверить только один тип клиента.

## 11. Соглашение о записи результатов

- Проверки, которые могут быть выполнены полностью, записываются как «Выполненные».
- Проверки, которые невозможно выполнить из-за экологических ограничений, четко обозначаются как «не выполнено + причина».
- Не путайте «должен пройти» с «проверено».
