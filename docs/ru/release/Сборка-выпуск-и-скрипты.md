## GitHub Действия
Текущий единый вход для публикации — `.github/workflows/release-all.yml`, метод запуска — `workflow_dispatch`, и он не будет запускаться автоматически.

### `release-all.yml`
- Цель: публикация настольных компьютеров в один клик + Service полноплатформенных продуктов.
- Сборка платформы: `Windows`, `macOS（dmg）`, `Linux`.
- Интерфейс `dist` сначала создается отдельно, а затем упаковывается и повторно используется каждым заданием платформы.
- Ввод:
  - `tag`: требуется
  - `ref`: `main` по умолчанию.
  - `prerelease`: по умолчанию `auto`, дополнительно `auto|true|false`.
- Поведение: выполнять только упаковку и публикацию, больше никаких тестовых шлюзов на стороне сервера.

## Выпуск списка продуктов

### Рабочий стол
- Windows: `CodexManager_<version>_x64-setup.exe`, `CodexManager-portable.exe`
- macOS: `CodexManager_<version>_aarch64.dmg`, `CodexManager_<version>_x64.dmg`
- Linux: `CodexManager_<version>_amd64.AppImage`, `CodexManager_<version>_amd64.deb`, `CodexManager-linux-portable.zip`

### Service
- Windows: `CodexManager-service-windows-x86_64.zip`
- macOS: `CodexManager-service-macos-arm64.zip`, `CodexManager-service-macos-x64.zip`
- Linux: `CodexManager-service-linux-x86_64.zip`
- Linux (пакет веб-тестирования): `CodexManager-web-linux-x86_64.zip`

### Тип выпуска
- Если `prerelease=auto`, `tag`, содержащий `-`, публикуется как pre-release.
- Если включен `prerelease=auto`, любая версия, не включающая `-`, будет выпущена как официальная версия.
- При использовании `prerelease=true|false` автоматическое решение на основе тега будет отменено.
- При повторном запуске того же `tag` метаданные Release будут синхронизированы в соответствии с текущим вводом.
- GitHub по-прежнему автоматически сопровождается `Source code (zip/tar.gz)`.

## `scripts/rebuild.ps1`
По умолчанию используется местная упаковка Windows; Режим `-AllPlatforms` вызывает рабочий процесс GitHub.

### Распространенные примеры
```powershell
# Локальная сборка Windows
pwsh -NoLogo -NoProfile -File scripts/rebuild.ps1 -Bundle nsis -CleanDist -Portable

# Запустить рабочий процесс выпуска (и загрузить артефакты)
pwsh -NoLogo -NoProfile -File scripts/rebuild.ps1 `
  -Все платформы `
  -GitRef главная `
  -ReleaseTag v0.1.9 `
  -GithubToken <token>

# Принудительная публикация как pre-release
pwsh -NoLogo -NoProfile -File scripts/rebuild.ps1 `
  -AllPlatforms -GitRef main -ReleaseTag v0.1.9-beta.1 -GithubToken <token> -Prerelease true
```

### Основные параметры
- `-Bundle nsis|msi`: `nsis` по умолчанию.
- `-NoBundle`: Только компиляция, без установочного пакета.
- `-CleanDist`: Очистка перед сборкой `apps/out`.
- `-Portable`: Портативная версия с дополнительным выходом.
- `-PortableDir <path>`: Выходной каталог портативной версии, по умолчанию `portable/`.
- `-AllPlatforms`: Запуск указанного рабочего процесса выпуска.
- `-GithubToken <token>`: токен GitHub; попробовать, если не прошел `GITHUB_TOKEN`/`GH_TOKEN`
- `-WorkflowFile <name>`: `release-all.yml` по умолчанию.
- `-GitRef <ref>`: ссылка на сборку рабочего процесса; текущая ветка или текущий тег по умолчанию
- `-ReleaseTag <tag>`: тег публикации; рекомендуется передавать его явно, если `-AllPlatforms`
- `-Prerelease <auto|true|false>`: `auto` по умолчанию.
- `-DownloadArtifacts <bool>`: `true` по умолчанию.
- `-ArtifactsDir <path>`: каталог загрузки артефактов, по умолчанию `artifacts/`.
- `-PollIntervalSec <n>`: интервал опроса, по умолчанию `10`.
- `-TimeoutMin <n>`: время ожидания в минутах, по умолчанию `60`.
- `-DryRun`: Распечатать только план выполнения.
