# Каталог документации

`docs/` — официальный каталог основной документации CodexManager.

Цели:
- хранить руководства по эксплуатации, заметки по релизам и правила сопровождения прямо в репозитории;
- помогать новым участникам быстро находить нужные документы без устной передачи контекста.

## Область ответственности
- Корневые `README.md` / `README.en.md`: обзор проекта и быстрый старт.
- Корневой `Журнал-изменений.md`: история версий и невыпущенные изменения.
- `report/*`: эксплуатация, устранение неполадок, совместимость и FAQ.
- `release/*`: сборка, упаковка, публикация и описание артефактов.

## С чего начать
- Актуальные заметки о релизах и невыпущенных изменениях смотрите в [Журнал-изменений.md](Журнал-изменений.md).
- Если не уверены, какой документ открыть первым, используйте таблицу ниже.

## Структура каталога

### `release/`
Заметки о релизах, планы отката, описание артефактов и инструкции по упаковке.

Операционные инструкции, заметки по устранению неполадок, отчёты о совместимости и FAQ.

## Правила по документации

### Документацию стоит коммитить, если она
- остаётся полезной для будущих участников,
- влияет на разработку, тестирование, выпуск или диагностику,
- или служит долгосрочным источником истины для проекта.

### Документацию не стоит коммитить, если это
- временный черновик,
- личные рабочие заметки,
- одноразовый промежуточный файл,
- или локальная запись эксперимента.

## Игнорируемые шаблоны
- `docs/**/*.tmp.md`
- `docs/**/*.local.md`

Не используйте эти суффиксы для официальной документации.

## Именование

```text
Долгоживущие документы: topic.md
Разовые отчёты: yyyyMMddHHmmssfff_topic.md
```

## Поддержка
- Добавляйте важные материалы в `docs/`, а не раздувайте README.
- Историю версий ведите в `Журнал-изменений.md`.
- Архитектурные заметки храните в `Архитектура.md`.
- Правила совместной работы храните в `Руководство-для-участников.md`.
- Подробности по невыпущенным изменениям фиксируйте в `Журнал-изменений.md`, а README оставляйте для навигации.
