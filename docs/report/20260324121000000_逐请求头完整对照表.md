# 逐请求头完整对照表

说明：以下以 `Responses` 相关请求为主，覆盖官方 Codex 最新源码与当前仓库的出站请求差异。

| 字段/行为 | Codex 官方 | 当前仓库 | 结论 |
| --- | --- | --- | --- |
| `Authorization` | `Bearer <token>`，由认证层注入 | `Bearer <token>`，由网关认证层注入 | 已对齐 |
| `Content-Type` | 有请求体时发送 `application/json` | 有请求体时发送 `application/json` | 已对齐 |
| `Accept` | `/responses` 流式请求为 `text/event-stream` | `/v1/responses` 流式请求为 `text/event-stream`；`/v1/responses/compact` 为 `application/json` | 大体对齐，compact 路径不同 |
| `User-Agent` | `codex_cli_rs/<env!("CARGO_PKG_VERSION")> (...)` | `codex_cli_rs/<运行时/数据库版本> (...)` | 差异，版本来源不同 |
| `originator` | 由 `codex_login::default_client::originator()` 统一生成，可被初始化/环境覆盖 | 出站固定为 `current_wire_originator()` | 差异，当前仓库的可配置值不直接影响出站 |
| `x-client-request-id` | 固定等于 `conversation_id` | 已基本对齐为原始 `conversation_id`；无 `conversation_id` 时沿用既有值 | 基本对齐，线程锚点不再直接污染外发请求 id |
| `session_id` | 固定等于 `conversation_id` | 已基本对齐为原始 `conversation_id`；无 `conversation_id` 时沿用既有值 | 基本对齐，会话标识已回到 conversation 维度 |
| `x-openai-subagent` | 由 `SessionSource` 映射 | 由 `SessionSource` 映射 | 已对齐 |
| `x-codex-beta-features` | 发送会话级 beta feature | 发送会话级 beta feature | 已对齐 |
| `x-codex-turn-metadata` | 发送 turn metadata | 发送 turn metadata | 已对齐 |
| `x-codex-turn-state` | 同一 turn 内持续回放 | 仅在稳定线程锚点存在且不与 `conversation_id` 冲突时保留；孤儿/切线程/冲突时清掉 | 差异，当前仓库更保守 |
| `ChatGPT-Account-ID` | 认证层注入 | 认证层注入 | 已对齐 |
| `x-openai-internal-codex-residency` | 按 residency 配置注入 | 按 residency 配置注入 | 已对齐 |
| `prompt_cache_key`（body） | 默认与 `conversation_id` 一致 | 初始一致，但网关会按线程锚点/强制键重写 | 差异，当前仓库多了一层重写 |
| 请求路径 | 代码层直接走 `responses` / `responses/compact`，由 base URL 拼接最终地址 | 外层暴露 `/v1/responses`，再由网关改写到 `.../backend-api/codex/...` | 差异，当前仓库多了一层路径转换 |
| `Connection` | 常规客户端不强制改写 | 对 `localhost/127.0.0.1/::1` 强制 `close` | 当前仓库特有的传输层保护 |

## 影响优先级

### P0

- `x-codex-turn-state`
- `prompt_cache_key`

这两项决定线程回放和账号粘性，仍然最容易造成“同一会话不稳定”的体感差异。

### P1

- `User-Agent`
- `originator`
- 请求路径改写

这三项会影响上游识别、埋点和后续兼容性，但通常不直接破坏一次请求。

### P2

- `x-openai-subagent`
- `x-codex-beta-features`
- `x-codex-turn-metadata`
- `ChatGPT-Account-ID`
- `x-openai-internal-codex-residency`

这些目前已经基本对齐，更多是确认而不是修复。
