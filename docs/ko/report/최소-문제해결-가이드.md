# 최소 문제 해결 매뉴얼

이 매뉴얼은 가장 일반적인 시작, 전달 및 모델 새로 고침 문제를 빠르게 찾는 데 사용됩니다.

## 먼저 살펴봐야 할 것

1. 먼저 [운영 및 배포 가이드](실행-및-배포-가이드.md)을 읽어보세요.
2. [FAQ 및 계정 라우팅 규칙](FAQ-및-계정-라우팅-규칙.md)을 다시 읽어보세요.

## 일반적인 문제 해결 순서

### 1. 서비스 시작 여부

- 데스크탑이나 서비스가 시작되었는지 확인하세요.
- 청취 주소, 포트 및 프록시 구성이 올바른지 확인하십시오.
- 로그를 확인하여 데이터베이스 연결 실패, 포트 점유 또는 권한 문제가 있는지 확인하세요.

### 2. 요청이 게이트웨이에 들어가는지 여부

- 클라이언트 요청이 실제로 현재 게이트웨이 주소로 이동하는지 확인합니다.
- 요청 로그에 해당 경로와 상태 코드가 있는지 확인하세요.
- 타사 클라이언트인 경우 API Key와 기본 URL이 올바르게 구성되었는지 확인하세요.

### 3. 사용 가능한 모델이나 계정이 있나요?

- 계정 상태가 비활성화, 만료 또는 한도에 도달했는지 확인하세요.
- 사용량을 새로 고치고 다시 시도하세요.
- 계정이 차단된 경우 먼저 계정 청산 입구로 이동하세요.

### 4. 스트리밍 또는 도구 예외

- `tool_calls`, SSE 종료 이벤트 및 필드 복원이 손실되었는지 확인합니다.

## 그래도 작동하지 않으면 어떻게 해야 하나요?

- README의 문서 항목을 직접 참조하세요.
- 조사를 계속하기 전에 오류 요청, 응답 및 요청 로그를 우선적으로 보관하십시오.
