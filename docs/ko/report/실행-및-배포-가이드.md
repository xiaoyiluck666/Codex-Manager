# 운영 및 배포 가이드

## 적용범위
- 데스크톱에서 처음 사용
- Service 버전은 독립적으로 실행됩니다.
- Docker 배포
- macOS 첫 번째 출시

## 빠른 시작
1. 데스크탑을 시작하고 "서비스 시작"을 클릭하십시오.
2. "계정 관리"에 들어가서 계정을 추가하고 인증을 완료하세요.
3. 콜백이 실패하면 콜백 링크를 붙여넣어 수동으로 구문 분석을 완료합니다.
4. 사용량을 새로 고치고 계정 상태를 확인하세요.

## 계정 가져오기 및 내보내기
- `批量导入`: 여러 개의 `.json/.txt` 파일을 선택하여 함께 가져옵니다.
- `按文件夹导入`: 데스크톱에서만 사용할 수 있습니다. 디렉터리를 선택한 후 `.json` 파일을 재귀적으로 스캔하여 일괄적으로 가져옵니다. 빈 파일은 자동으로 건너뜁니다.
- `导出用户`: 디렉터리를 선택한 후 "계정당 하나의 JSON 파일"을 클릭하여 내보내 백업 및 마이그레이션을 용이하게 합니다.

## Service 버전
1. 릴리스에서 `CodexManager-service-<platform>.zip`을 다운로드하고 압축을 풉니다.
2. `codexmanager-start`을 시작하는 것이 좋습니다. 프로세스는 서비스 + 웹을 시작하고 콘솔 `Ctrl+C`에서 직접 닫을 수 있습니다.
3. `codexmanager-web`을 실행하면 동일한 디렉토리에 자동으로 `codexmanager-service`이 표시되고 브라우저가 열립니다.
4. 또는 `codexmanager-service`을 먼저 시작한 다음 `codexmanager-web`을 시작하십시오.
5. 기본 주소: 서비스 `localhost:48760`, Web UI `http://localhost:48761/`.
6. 닫기: `http://localhost:48761/__quit`을 방문하세요. 웹 서비스가 자동으로 시작된 경우 함께 닫으려고 시도합니다.
7. 직접 역방향 프록시를 수행하거나 프런트엔드 정적 리소스를 분할 배포해야 하는 경우 `/api/runtime` 및 `/api/rpc`을 동시에 전달해야 합니다. 정적 파일만 호스팅하면 관리 페이지가 제대로 작동하지 않습니다.

## Docker 배포

### GitHub 패키지/GHCR
- 릴리스가 출시된 후 `codexmanager-service` 및 `codexmanager-web` 이미지가 GitHub 패키지(GHCR)에 동시에 푸시됩니다.
- 해당 릴리스 태그를 당기기만 하면 됩니다. 예: `docker pull ghcr.io/qxcnm/codexmanager-service:v0.1.15`
- 저장소의 [`docker/docker-compose.release.yml`](../../../docker/docker-compose.release.yml) 역시 GHCR을 직접 참조하므로, 사용 전에 `CODEXMANAGER_RELEASE_TAG`을 설정하세요.
- 예: `CODEXMANAGER_RELEASE_TAG=v0.1.15 docker compose -f docker/docker-compose.release.yml up -d`

### 방법 1: docker compose
``배쉬
docker compose -f docker/docker-compose.yml up --build
````

브라우저 열기: `http://localhost:48761/`

### 방법 2: 별도로 빌드하고 실행
``배쉬
#서비스
docker build -f docker/Dockerfile.service -t codexmanager-service .
docker run --rm -p 48760:48760 -v codexmanager-data:/data \
  -e CODEXMANAGER_RPC_TOKEN=replace_with_your_token \
  코덱스 관리자 서비스

# 웹 (서비스 접근 필요)
docker build -f docker/Dockerfile.web -t codexmanager-web .
docker run --rm -p 48761:48761 \
  -v codexmanager-data:/data \
  -e CODEXMANAGER_WEB_NO_SPAWN_SERVICE=1 \
  -e CODEXMANAGER_SERVICE_ADDR=host.docker.internal:48760 \
  -e CODEXMANAGER_RPC_TOKEN=replace_with_your_token \
  코덱스 관리자 웹
````

- 참고: Web 비밀번호, 설정 항목, 캐시 모델 목록 및 기타 상태가 서비스와 일치하도록 하려면 `codexmanager-web` 및 `codexmanager-service`이 동일한 `/data` 볼륨을 공유해야 합니다.

## macOS 첫 시작
- 현재 macOS 릴리스 제품은 Apple 개발자 계정을 사용하여 공증되지 않았으므로 브라우저에서 처음 다운로드한 후 Gatekeeper에서 "손상됨" 메시지를 표시하거나 열기를 거부할 수 있습니다.
- macOS `dmg` 릴리스에는 `Open CodexManager.command` 및 `README-macOS-first-launch.txt`이 내장되어 있습니다.
- 먼저 `CodexManager.app`을 "애플리케이션"으로 드래그한 후 스크립트를 더블클릭하여 첫 번째 릴리스를 완료하는 것이 좋습니다.
- 직접 실행할 수도 있습니다.

``배쉬
xattr -dr com.apple.quarantine /Applications/CodexManager.app
````

- 그래도 차단되어 있는 경우 `CodexManager.app` 항목을 다시 "우클릭 -> 열기"를 실행해 주세요.
