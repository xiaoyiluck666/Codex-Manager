# 출시 및 산출물 안내

## 적용범위

이 문서는 현재 저장소의 통합 릴리스 진입점, 산출물 목록, 수동 트리거 매개변수, 대표적인 문제 해결 경로를 설명합니다.

## 통일출판입구

현재 유일한 게시 작업 흐름은 다음과 같습니다.

- `.github/workflows/release-all.yml`
- 프론트엔드 `dist`을 먼저 별도로 구축한 후 패키징 및 작업 재사용을 위해 각 플랫폼에 배포할 예정입니다.

트리거 방법:

- `workflow_dispatch`만 지원
- 기본적으로 푸시/풀 요청으로 자동 트리거되지 않습니다.

핵심 입력:

- `tag`: 릴리스 태그, 필수
- `ref`: 기준 브랜치 또는 커밋 구축, 기본값 `main`
- `prerelease`: `auto | true | false`
- 참고: 워크플로는 패키징 및 게시만 담당하며 더 이상 서버 측 테스트 게이트를 포함하지 않습니다.

GitHub 릴리스는 여전히 자동으로 다음과 함께 제공됩니다.

- `Source code (zip)`
- `Source code (tar.gz)`

## pre-release 규칙

- `prerelease=auto` 및 `tag`에는 `-`이 포함됩니다: pre-release로 게시됨
- `prerelease=auto` 및 `tag`에는 `-`이 포함되지 않습니다: 공식 버전으로 출시됨
- `prerelease=true|false`: 자동판정 강제 무시
- 동일한 `tag`을 다시 실행하면 릴리스 메타데이터가 이 입력을 기반으로 다시 동기화됩니다.

## 로컬 트리거 항목

Windows 로컬 보조 스크립트:

- `scripts/rebuild.ps1`

일반적인 사용법:

``파워셸
pwsh -NoLogo -NoProfile -파일 스크립트/rebuild.ps1 `
  -모든 플랫폼`
  -GitRef 메인`
  -ReleaseTag v0.1.9`
  -GithubToken <token>
````

일반적으로 사용되는 매개변수:

- `-AllPlatforms`: 트리거 `release-all.yml`
- `-ReleaseTag`: 태그 게시
- `-GitRef`: 참조 빌드
- `-Prerelease`: pre-release 상태를 명시적으로 지정합니다.
- `-DownloadArtifacts`: 트리거 후 빌드 아티팩트 다운로드 여부

## 플랫폼 차이점 설명

### Windows

- 설치버전과 포터블 버전을 동시에 제작
- 휴대용 버전은 현재 단일 `exe`이며, zip의 추가 레이어는 포함되지 않습니다.

### macOS

- 현재 제품은 `dmg`입니다.
- Apple 개발자 계정은 공증되지 않았기 때문에 첫 시작 시 Gatekeeper에 의해 차단될 수 있습니다.
- `dmg` 포함:
  - `Open CodexManager.command`
  - `README-macOS-first-launch.txt`

### Linux

- 현재 데스크탑에서는 `AppImage` 및 `deb`를 사용할 수 있습니다.
- Service 버전이 압축된 패키지 형태로 출시됩니다.

## 출시 전 확인을 권장합니다

1. `scripts/bump-version.ps1`을 통해 버전번호가 동기화 되었는지 확인하세요.
2. `변경-이력.md`이 업데이트되었는지 확인하세요.
3. 데스크톱 프런트엔드 빌드가 통과되었는지 확인합니다: `pnpm -C apps run build`
4. 핵심 테스트를 통과했는지 확인하십시오: `pnpm -C apps run test`, `cargo test --workspace`
5. 게이트웨이 프로토콜 변경 사항이 포함된 경우 추가 `scripts/tests/gateway_regression_suite.ps1`

## 일반적인 오류 문제 해결

### 프런트엔드 아티팩트 누락

우선순위 확인:

- `apps/out/` 정상적으로 구축이 가능한지 여부
- 워크플로의 프런트엔드 빌드 단계가 성공적으로 완료되었는지 여부

### 출시 메타데이터가 잘못되었습니다.

우선순위 확인:

- `tag`에는 `-`이 포함됩니까?
- `prerelease` 자동 판단이 명시적으로 무시되는지 여부

### macOS 제품을 다운로드할 수는 있지만 처음 열 수는 없습니다.

이는 현재 공증되지 않은 상태에서 예상되는 제한 사항이며 워크플로 빌드 실패가 아닙니다.

처리 방법:

1. 먼저 `CodexManager.app`을 "응용 프로그램"으로 드래그합니다.
2. `Open CodexManager.command`을 두 번 클릭합니다.
3. 또는 다음을 실행합니다.

``배쉬
xattr -dr com.apple.quarantine /Applications/CodexManager.app
````
