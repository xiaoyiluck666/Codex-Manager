## GitHub 작업
현재 통합출판 입구는 `.github/workflows/release-all.yml`이고, 발동방식은 `workflow_dispatch`이며, 자동으로 발동되지 않습니다.

### `release-all.yml`
- 목적: 원클릭 퍼블리싱 데스크탑 + Service 풀 플랫폼 제품
- 빌드 플랫폼: `Windows`, `macOS（dmg）`, `Linux`
- 프런트엔드 `dist`는 먼저 별도로 구축된 다음 각 플랫폼 작업별로 패키징되어 재사용됩니다.
- 입력:
  - `tag`: 필수
  - `ref`: 기본값 `main`
  - `prerelease`: 기본값 `auto`, 선택 `auto|true|false`
- 동작: 패키징 및 게시만 수행하고 더 이상 서버 측 테스트 게이트는 수행하지 않습니다.

##출시제품목록

### 데스크탑
- Windows: `CodexManager_<version>_x64-setup.exe`, `CodexManager-portable.exe`
- macOS: `CodexManager_<version>_aarch64.dmg`, `CodexManager_<version>_x64.dmg`
- Linux: `CodexManager_<version>_amd64.AppImage`, `CodexManager_<version>_amd64.deb`, `CodexManager-linux-portable.zip`

### Service
- Windows: `CodexManager-service-windows-x86_64.zip`
- macOS: `CodexManager-service-macos-arm64.zip`, `CodexManager-service-macos-x64.zip`
- Linux: `CodexManager-service-linux-x86_64.zip`
- Linux(웹 테스트 패키지): `CodexManager-web-linux-x86_64.zip`

### 출시 유형
- `prerelease=auto`, `-`을 포함하는 `tag`가 pre-release로 게시되는 경우.
- `prerelease=auto`이 포함되면 `-`이 포함되지 않은 모든 버전이 정식 버전으로 출시됩니다.
- `prerelease=true|false`을 사용하면 태그에 따른 자동 판단이 무시됩니다.
- 동일한 `tag`을 다시 실행하면 릴리스 메타데이터가 현재 입력에 따라 동기화됩니다.
- GitHub은 여전히 ​​`Source code (zip/tar.gz)`과 함께 자동으로 제공됩니다.

## `scripts/rebuild.ps1`
기본값은 로컬 Windows 패키징입니다. `-AllPlatforms` 모드는 GitHub 워크플로를 호출합니다.

### 일반적인 예
``파워셸
# 로컬 Windows 빌드
pwsh -NoLogo -NoProfile -파일 스크립트/rebuild.ps1 -Bundle nsis -CleanDist -Portable

# 릴리스 워크플로 트리거(및 아티팩트 다운로드)
pwsh -NoLogo -NoProfile -파일 스크립트/rebuild.ps1 `
  -모든 플랫폼`
  -GitRef 메인`
  -ReleaseTag v0.1.9`
  -GithubToken <token>

# pre-release로 강제 게시
pwsh -NoLogo -NoProfile -파일 스크립트/rebuild.ps1 `
  -AllPlatforms -GitRef 메인 -ReleaseTag v0.1.9-beta.1 -GithubToken <token> -프리릴리즈 true
````

### 주요 매개변수
- `-Bundle nsis|msi`: 기본값 `nsis`
- `-NoBundle`: 컴파일만 가능, 설치 패키지 없음
- `-CleanDist`: 빌드 전 정리 `apps/out`
- `-Portable`: 추가 출력 휴대용 버전
- `-PortableDir <path>`: 휴대용 버전 출력 디렉터리, 기본값 `portable/`
- `-AllPlatforms`: 지정된 릴리스 워크플로 트리거
- `-GithubToken <token>`: GitHub 토큰; 통과하지 못한 경우 시도 `GITHUB_TOKEN`/`GH_TOKEN`
- `-WorkflowFile <name>`: 기본값 `release-all.yml`
- `-GitRef <ref>`: 워크플로 빌드 참조; 기본 현재 분기 또는 현재 태그
- `-ReleaseTag <tag>`: 태그 게시 `-AllPlatforms`의 경우 명시적으로 전달하는 것이 좋습니다.
- `-Prerelease <auto|true|false>`: 기본값 `auto`
- `-DownloadArtifacts <bool>`: 기본값 `true`
- `-ArtifactsDir <path>`: 아티팩트 다운로드 디렉터리, 기본값 `artifacts/`
- `-PollIntervalSec <n>`: 폴링 간격, 기본값 `10`
- `-TimeoutMin <n>`: 제한 시간(분), 기본값 `60`
- `-DryRun`: 실행 계획만 인쇄
